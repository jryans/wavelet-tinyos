Commands

  • void registerListener(void) Each module that listens to the requestDone event should call this method during startup to ensure that no pointers are freed until all modules are done with them.

  • result_t request (void) Requests big pack data of a certain type.

  • void free(void) When an application is done with the data, it must call free so that internal structures can be deallocated.

  • void requestDone (void *mainBlock, result_t result) Once the request is complete, the requester is given a pointer to the main data block.

Commands - Details

request

result_t request(void)

Requests big pack data of a certain type. If another request is already being processed, this will return FAIL and do nothing. Otherwise, it returns SUCCESS and begins the request.

Events - Details

requestDone

void requestDone(void *mainBlock, result_t result)

Once the request is complete, the requester is given a pointer to the main data block. This pointer is only valid when result is SUCCESS.
